## 1. Les 3 risques qui peuvent tuer le projet

### ✅ #1 — Gap HDS « Activité 5 » — **RÉSOLU (correction post-audit 2026-06-25)**
> ⚠️ **L'audit a surévalué ce risque** : il s'est appuyé sur la formulation prudente de `CLAUDE.md` au lieu de la confirmation écrite obtenue d'AZNetwork le 2026-06-12. **Le gap est en réalité fermé.**

- AZNetwork est certifié HDS sur les **6 activités (1-2-3-4-5-6)**, activité 5 (exploitation du SI de santé) **incluse**, couverte par l'infogérance jusqu'à la couche système. Confirmé par écrit (descriptif AZNetwork 2026-06-12) ; certificats en main (`AZNETWORK-Certificat-HDS-…2026-2028.pdf` + ISO 27001) ; contrat + attestation hébergement santé signés.
- PsyLib = **éditeur exploitant son propre SI** chez un hébergeur certifié → **pas de certification HDS propre requise**. Accès prod via bastion Wallix (tracé) = renfort.
- **Seule règle qui demeure** : ne jamais écrire « **PsyLib** est certifié HDS » — formulation correcte : *« données hébergées en France chez un hébergeur certifié HDS »* (AZNetwork).
- Hygiène résiduelle (pas un blocage) : conserver contrat + attestation signés dans les dossiers conformité.

→ **Conséquence** : le risque n°1 « existentiel » disparaît. Le vrai risque structurel n°1 devient le **SPOF infra** (#2 ci-dessous).

### 🔴 #2 — Point de défaillance unique infra (opérationnel)
Un **seul VPS** héberge API + PostgreSQL + Redis + Keycloak + LiveKit. Sa chute = **arrêt total**. Backups existants mais **restauration jamais testée**, pas de RTO/RPO, pas de standby.

**Action (M3-M6)** : externaliser PostgreSQL (base managée HDS ou VM + réplication) ; documenter et **tester** un plan de reprise.

### 🔴 #3 — Bus factor = 1 (humain & business)
- **Déploiement 100% manuel** (tar/scp/build local). Le pipeline `deploy.yml` versionné est **cassé** (cible l'ancien VPS OVH décommissionné, healthcheck sur `/health` au lieu de `/api/v1/health`).
- **Founder solo** = goulot d'étranglement garanti au-delà de M9-M12.

**Action** : réparer/formaliser le déploiement (script versionné idempotent + Vercel via CI) ; automatiser le support N1 ; recruter un CS à mi-temps dès ~20 000€ MRR.

## 2. Volet technique

### Ce qui est excellent (à préserver)
- Typage strict réel : **0 `any`** sur 247 fichiers API.
- Chiffrement **AES-256-GCM** : IV aléatoire/message, authTag vérifié, **versioning de clé + rotation progressive** (`common/encryption.service.ts`).
- Isolation multi-tenant **systématique** + audit `DECRYPT` à chaque déchiffrement (`patients.service.ts`).
- Sécurité web solide : webhooks Stripe signés, uploads validés par magic bytes, CORS sans wildcard, CSP stricte, SQL paramétré, secrets hors-code.
- **92 index Prisma** pertinents, pagination, ~89 fichiers de tests.

## 3. Volet sécurité & conformité

- **Posture sécurité applicative : 8,0/10.** Architecture défensive mature.
- **RGPD : 8,5/10.** Consentement IA explicite avant tout envoi LLM, droit à l'effacement réel avec archivage de preuve, Sentry sans PII.
- **HDS : 4,0/10.** Bloquant = activité 5 (cf. §1).

### Écarts notables
- `GLOBAL_AUTH_GUARD` **désactivé par défaut** → modèle fail-open si un futur contrôleur oublie `@UseGuards`. (Aucun endpoint sensible exposé aujourd'hui.)
- Secrets en `.env` sur le VPS (pas de coffre) → compromission serveur = compromission du corpus.
- `audit_logs` **altérables** (pas de WORM/chaînage) → preuve d'accès fragilisée juridiquement.
- CSP `'unsafe-inline'` sur `script-src` (Crisp/PostHog) ; rate-limit global non différencié sur `/auth` et `/ai`.
- `network.service` : `referral`/`networkGroup` chargés sans scope tenant (feature collaborative, risque résiduel faible).

## 5. Volet marketing

- **Note : 6,2/10.** Produit fort, exécution marketing en amorçage.
- **Marché** : ~30 000 psys libéraux FR, profession la plus active en téléconsultation (25,9 %), 29,9 % perdent 10-20h/mois en admin. E-santé mentale FR : TCAM 18,5 %. Catalyseur MonSoutienPsy (tiers payant oct. 2026).
- **Différenciation forte** : seul acteur FR combinant IA native + Scribe audio + visio + comptabilité FEC + HDS à 40€.
- **Cible de déplacement #1** : Terapiz (~40€, même prix que Pro, sans IA ni compta).
- **Frein principal = confiance** : preuve sociale quasi nulle (« 7+ fondateurs », aucun avis tiers), micro-entreprise solo non humanisée, acquisition mono-canal (cold email).

### Pricing — débat ouvert
- Audit marketing : Pro sous-pricé → Solo 29€ / Pro 49€ (+22 % MRR à base constante).
- Audit croissance : ne pas toucher avant M12 / 300 clients + NPS>50.
- **Synthèse** : garder les prix jusqu'à preuve sociale + NPS solides, puis tester +5€/+8€ en grandfathering.
